# Agent Eval Digest — 2026-04-05

## TL;DR
Open models like GLM-5 and MiniMax M2.7 are now performing on par with closed frontier models for agent tasks such as tool use and instruction following, potentially allowing Connectly to reduce costs and latency in our Test Framework without sacrificing quality. Additionally, the pydantic-ai release introduces new evaluation infrastructure that could enhance our ability to run online evals, making it easier to integrate deterministic testing for multi-turn conversations. These developments matter because they address key challenges in scaling agent evaluations and tool use testing at Connectly.

## Top Findings

### Evaluation Frameworks & Benchmarks

### [Open Models have crossed a threshold](https://blog.langchain.com/open-models-have-crossed-a-threshold/)
*Source: rss | 2026-04-02*

This blog post from LangChain highlights how open-source models like GLM-5 and MiniMax M2.7 now rival closed models in agent-specific tasks, including file operations, tool use, and instruction following, backed by their internal evals that show significant cost and latency reductions. What's novel is the demonstration that these models can handle real-world agent scenarios effectively, which could inspire Connectly to switch to cheaper open models in our Test Framework for running simulated personas, potentially speeding up iteration on multi-turn conversations while maintaining accuracy in outcomes like tool calls. For Connectly, this means exploring integration of these models to enhance our regression testing, ensuring we catch subtle issues in prompt updates without increasing compute costs.

### [pydantic/pydantic-ai v1.74.0](https://github.com/pydantic/pydantic-ai/releases/tag/v1.74.0)
*Source: github | 2026-03-31*

This GitHub release for pydantic-ai introduces features like online evaluation infrastructure and enhanced handling of user prompts with metadata, aimed at improving how AI models are tested in structured environments. What's interesting is the addition of tools for running evals that could support deterministic testing, such as separating metadata from model inputs, which aligns with Connectly's use of mock data in our Test Framework to simulate personas and evaluate agent responses. For Connectly, this could mean adopting pydantic-ai's eval infrastructure to bolster our multi-turn dialogue testing, making it easier to automate checks for tool use and conversation outcomes, thus improving our regression suite for prompt changes.
