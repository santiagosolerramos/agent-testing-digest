# Agent Eval Digest — 2026-04-05

## TL;DR
Open models like GLM-5 and MiniMax M2.7 are now competitive with closed frontier models for agent tasks, offering cost and latency savings that could let Connectly swap in cheaper backends for our WhatsApp agents without sacrificing performance. ## Top Findings

### Evaluation Frameworks & Benchmarks

#### Open Models have crossed a threshold
*Source: rss | 2026-04-02*

This blog post from LangChain highlights how open-source models like GLM-5 and MiniMax M2.7 now perform on par with proprietary models in agent-specific tasks such as file operations, tool use, and instruction following, based on their internal evals. What's novel is the demonstration that these models achieve this at lower costs and latency, making them viable alternatives for production agents. For Connectly, this means we could integrate these models into our Test Framework to reduce operational expenses while maintaining agent reliability, especially for multi-turn WhatsApp conversations involving tool calls—though we'd need to verify they handle edge cases like ambiguity or escalations as well as our current setups.

#### pydantic/pydantic-ai v1.74.0
*Source: github | 2026-03-31*

This GitHub release for Pydantic-AI introduces features like an online evaluation infrastructure for pydantic-evals and enhanced handling of user prompts with metadata, improving how models process and evaluate inputs in agent workflows. The novelty lies in the seamless integration for running evals in real-time, which could help automate testing of tool use and response accuracy without flooding our pipelines with false positives. For Connectly's Test Framework, this is directly applicable for building more robust regression suites, as it allows us to mock and evaluate personas more efficiently in multi-turn simulations—potentially catching subtle prompt regressions by tracking metadata not passed to the model.
