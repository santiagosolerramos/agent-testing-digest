# Agent Eval Digest — 2026-04-05

## TL;DR
Open models like GLM-5 and MiniMax M2.7 are now matching the performance of closed frontier models on core agent tasks such as tool use and instruction following, offering significant cost and latency advantages. This development is particularly relevant for Connectly, as it could allow us to enhance our Test Framework by incorporating these efficient models for synthetic user simulations and evaluations, potentially reducing expenses while maintaining high-fidelity testing for multi-turn conversations. However, we'd need to verify their reliability in handling edge cases specific to WhatsApp agents, like language diversity and tool calls.

### Open Models have crossed a threshold
[https://blog.langchain.com/open-models-have-crossed-a-threshold/](https://blog.langchain.com/open-models-have-crossed-a-threshold/)
*Source: rss | 2026-04-02*

This LangChain blog post details how open-source models like GLM-5 and MiniMax M2.7 have reached parity with closed models on agent-specific tasks, including file operations, tool use, and instruction following, based on their internal evaluations. What's novel is the empirical data showing these models deliver equivalent results at a fraction of the cost and latency, making them practical for real-world applications. For Connectly's Test Framework, this means we could swap in these open models to run more frequent and scalable simulations of personas in multi-turn conversations, especially for testing tool use in deterministic environments, but we'd have to adapt our evals to ensure they handle our specific requirements like no hallucinations during order resolutions or escalations.
