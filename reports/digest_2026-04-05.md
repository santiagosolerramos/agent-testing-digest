# Agent Eval Digest — 2026-04-05

## TL;DR
Open models like GLM-5 are now competitive with closed models on key agent tasks such as tool use, potentially allowing Connectly to reduce costs and latency in our Test Framework without sacrificing performance. Additionally, the latest pydantic-ai release introduces online evaluation infrastructure, which could enhance our ability to run deterministic, multi-turn conversation tests more efficiently. These developments highlight opportunities to scale agent evaluations affordably while maintaining reliability in handling WhatsApp interactions.

## Top Findings

### Evaluation Frameworks & Benchmarks

#### Open Models have crossed a threshold
*Source: rss | 2026-04-02*

This blog post from LangChain discusses how open models like GLM-5 and MiniMax M2.7 now perform on par with closed frontier models in agent tasks such as file operations, tool use, and instruction following, backed by their internal evaluations that show significant cost and latency reductions. What's novel is the demonstration that these models can handle real-world agent scenarios effectively, making them viable alternatives for production-grade testing. For Connectly, this means we could integrate these open models into our Test Framework to simulate user interactions more affordably, potentially improving the speed of regression testing for multi-turn WhatsApp conversations while ensuring reliable tool use without the high costs of proprietary models.

#### pydantic/pydantic-ai v1.74.0
*Source: github | 2026-03-31*

This GitHub release for pydantic-ai introduces features like online evaluation infrastructure and enhanced handling of user prompts with metadata, enabling more structured and automated testing of AI models. The addition of tools for evals makes it easier to set up deterministic tests, which is particularly useful for verifying agent behaviors in controlled environments. For Connectly, this could directly enhance our Test Framework by providing a plug-and-play way to implement robust evaluation pipelines for personas and multi-turn dialogues, helping us catch subtle regressions in prompt updates or tool calls more effectively and ensuring our WhatsApp agents handle escalations accurately.
