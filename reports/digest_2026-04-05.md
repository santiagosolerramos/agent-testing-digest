### Evaluation Frameworks & Benchmarks
This section covers new tools and benchmarks that could integrate with Connectly's Test Framework for more robust agent evaluation.

#### Open Models have crossed a threshold
*Source: rss | 2026-04-02*

This blog post from LangChain discusses how open models like GLM-5 and MiniMax M2.7 now perform on par with closed frontier models in agent tasks such as file operations, tool use, and instruction following, offering lower costs and faster latency. What's novel is the empirical evidence from their evals, showing that these models handle real-world agent scenarios effectively, which could inspire us to benchmark our agents against open alternatives for cost savings. For Connectly, this means we could swap in these models for our WhatsApp agents to reduce operational expenses while maintaining accuracy in tool calls and multi-turn interactions, but we'd need to adapt our Test Framework to include these benchmarks to verify no regressions in edge cases like ambiguity handling.

#### pydantic/pydantic-ai v1.74.0
*Source: github | 2026-03-31*

This GitHub release for pydantic-ai introduces features like an online evaluation infrastructure and enhanced handling of user prompts with metadata, making it easier to build and run structured evaluations for AI systems. What's interesting is the addition of tools for seamless integration of evals into CI/CD pipelines, which addresses pain points in scaling tests without false positives. For Connectly, this is directly applicable to our Test Framework—we could adopt this infrastructure to automate persona-based simulations and tool-use evaluations, ensuring our agents' responses remain consistent across model updates and helping catch subtle regressions in multi-language conversations.
